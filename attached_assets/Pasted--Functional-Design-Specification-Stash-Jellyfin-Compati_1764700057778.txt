# Functional Design Specification – Stash → Jellyfin-Compatible Proxy Server (v1)

**Project Name:** `stash-jellyfin-proxy`  
**Version:** 1.0 (Server-only, no UI)  
**Target Platform:** Linux Mint (Python 3.10+) → future Docker migration  
**Goal:** Expose Stash’s entire library as a Jellyfin-compatible media server so that Infuse (and any Jellyfin client) can browse and stream content directly from Stash without Jellyfin itself being installed.

### Core Business Requirements
1. Act as a drop-in Jellyfin media server endpoint (Infuse connects to it exactly like it connects to Jellyfin).
2. All media metadata and file access are sourced 100 % from an existing Stash instance via its GraphQL API and direct file paths.
3. No duplication of media files or metadata storage.
4. Must be a single standalone Python script for v1 (easy migration to Docker + optional FastAPI/Flask UI in v2).
5. Authentication: support Jellyfin’s token-based auth (simple shared API key is acceptable for v1 home use).

### High-Level Architecture (v1)
```
Infuse (iOS/tvOS) 
    ↓  HTTP/HTTPS (Jellyfin protocol)
stash-jellyfin-proxy (Python script running an ASGI/WSGI server)
    ↓  GraphQL + direct file serving
Stash instance (stash.feldorn.com:443)
    ↓  filesystem
Actual video files on Linux host
```

### Functional Components

#### 1. Configuration & Secrets Management
- Single config file: `/home/chris/.scripts.conf` (same file used by existing `stash_scan.py`)
- The script must `exec()` this file at startup (exactly like the reference script) to load:
  ```python
  STASH_URL = "https://stash.feldorn.com"
  STASH_API_KEY = "eyJhbGciOiJIUz..."
  MEDIA_ROOTS = ["/data/Video-A", "/data/Video-B", "/data2/Video-B"]  # all paths Stash knows
  PROXY_BIND = "0.0.0.0"
  PROXY_PORT = 8096
  PROXY_API_KEY = "infuse12345"          # simple shared key for Infuse → proxy auth (v1)
  ```
- No hardcoded secrets in the script.

#### 2. HTTP Server
- Use **Hypercorn** (ASGI) or **uvicorn** with Starlette/FastAPI for maximum v2 compatibility.
- Listen on configurable `PROXY_BIND:PROXY_PORT` (default 8096).
- Support both HTTP and HTTPS (self-signed cert is acceptable for v1; cert paths loaded from config if present).

#### 3. Jellyfin Emulation Endpoints (minimum viable set for Infuse)
Implement only the endpoints Infuse actually uses:

| Path                                 | Jellyfin Purpose                  | Implementation Strategy |
|--------------------------------------|-----------------------------------|-------------------------|
| `/`                                  | Server discovery                  | Return Jellyfin `ServerInfo` JSON |
| `/System/Info`                       | Server capabilities               | Hard-coded reasonable values |
| `/Users/AuthenticateByName`          | Login (username/password or key)  | Accept `PROXY_API_KEY` as password → return fixed user + token |
| `/Items`                             | Library root & folder browsing    | Map Stash studios → Jellyfin collections; Stash folders → virtual folders |
| `/Items?parentId=...`                | Browse folder/studio              | Query Stash `findScenes` / `findGalleries` with path/studio filters |
| `/Items/{id}`                        | Item details                      | Return merged Stash scene/gallery metadata in Jellyfin format |
| `/Videos/{id}/stream.*`             | Direct stream                     | 303 redirect OR proxy to actual file path (X-Accel-Redirect style or direct sendfile) |
| `/Images/...`                        | Thumbnails, covers, sprites       | Proxy or redirect to Stash-generated images |
| `/PlaybackInfo`                      | Transcode decision               | Always return direct play (Stash files are already compatible) |

#### 4. Stash Interaction Layer
- Reuse exact URL/header pattern from the provided script:
  ```python
  GRAPHQL_URL = f"{STASH_URL}/graphql-local"
  HEADERS = {"ApiKey": STASH_API_KEY, "Content-Type": "application/json"}
  ```
- Core queries needed:
  - `findScenes` (with filters: paths, studios, tags, etc.)
  - `findStudios`
  - `findDirectories` / `findScenes` with `path` filter for folder browsing
  - `findScene` / `findGallery` by ID for detail view
  - `configuration` → get all configured paths for validation

#### 5. File Serving Strategy (Critical for Performance)
Two options (choose the fastest that works on Linux Mint):
- Preferred: **X-Accel-Redirect** style via local Nginx in front → proxy does 307 redirect to internal location that serves files directly (zero Python overhead).
- Fallback: Direct streaming with `sendfile()` or range-aware `aiofiles` (acceptable for v1 home use).

#### 6. Metadata Mapping (Stash → Jellyfin)
| Stash field           | Jellyfin field            | Notes |
|-----------------------|---------------------------|-------|
| title                 | Name                      |       |
| studio.name           | Studio                    |       |
| performers[].name     | People (Actor)            |       |
| tags[].name           | Genres + Tags             |       |
| date                  | PremiereDate / DateCreated|       |
| paths.file            | Path (for direct play)    |       |
| sceneMarkers          | Chapters (optional v1)    | can be omitted initially |
| cover / sprite URLs   | Image tags                | proxy them |

#### 7. Authentication Flow (v1 simplicity)
- Infuse sends POST `/Users/AuthenticateByName` with username (any) + password = `PROXY_API_KEY`
- Return fixed user ID + access token (static UUID or random once at startup)
- All subsequent requests must include header `X-Emby-Token: <token>`
- No user management, no database.

#### 8. Logging & Operational Requirements
- Log to syslog with program name `stash-jellyfin-proxy` (same mechanism as reference script)
- Support `--debug` flag for verbose logging
- Graceful shutdown on SIGTERM/SIGINT (same signal handlers as reference script)

#### 9. Future-Proofing for v2 (Docker + UI)
- All config via the `.scripts.conf` exec pattern (easily mountable as volume)
- HTTP server uses ASGI (Hypercorn/uvicorn) → same server can power FastAPI UI later
- No global state that prevents multiple instances
- All paths absolute → works when container root differs from host

### Deliverables for AI Implementation
A single file: `stash_jellyfin_proxy.py`
- Must run with: `python3 stash_jellyfin_proxy.py [--debug]`
- Must load `/home/chris/.scripts.conf` exactly like the provided `stash_scan.py`
- Must expose a working Jellyfin server on port 8096 that Infuse can connect to and browse/stream all Stash content

This specification is deliberately limited to functional behavior and environmental constraints only – all coding decisions (FastAPI vs Starlette, exact redirect vs proxy streaming, etc.) are left to the implementing AI.
